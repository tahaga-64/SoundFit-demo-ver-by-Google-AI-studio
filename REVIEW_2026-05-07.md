## 2. 重要度つきレビュー指摘

### [中] `setMessages([...messages, newMessage])` は連続送信時に取りこぼしリスク
- 該当: `ChatRoom` の `handleSend`。
- 現状はクロージャで捕捉された `messages` を使って配列展開しており、短時間に連打や非同期更新が重なると、最新状態を反映しきれない可能性があります。
- 提案: 関数型更新 `setMessages(prev => [...prev, newMessage])` に変更すると安全です。

### [中] `onDragEnd` の引数が `any` で型安全性が弱い
- 該当: `const handleDragEnd = (_: any, info: any) => { ... }`。
- Motionのイベント型を明示したほうが、将来的なリファクタや補完精度の面で有利です。
- 提案: `PanInfo` 等の型を利用し、`any` を除去。

### [低] `SwipeCard` の props に `key?: string` が含まれている
- 該当: `SwipeCard` の型定義。
- React の `key` はコンポーネントに props として渡されない特殊属性です。型に含めると誤解を生みます。
- 提案: `key?: string` を削除。

### [低] 画像要素に `alt` がない箇所がある
- 該当: チャットヘッダーのプロフィール画像。
- 別箇所の画像では `alt` が設定済みで、ここだけ一貫性がありません。
- 提案: `alt={profile.name}` などを付与し、アクセシビリティを改善。

### [低] `MusicProfile.compatibility` が optional のため表示上 `undefined%` になり得る
- 該当: `{profile.compatibility}% VIBE MATCH`。
- 現在のダミーデータでは値が入っていますが、将来API連携で欠損時にUI崩れが起こり得ます。
- 提案: `profile.compatibility ?? 0` などのフォールバック。

## 3. 優先度順の改善提案
1. `setMessages` を関数型更新へ変更（安全性）。
2. `onDragEnd` の `any` を具体型へ置換（保守性）。
3. `key` props型の削除、`alt` 追加、`compatibility` フォールバック（品質底上げ）。
