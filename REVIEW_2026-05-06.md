## 2) 確認した注意点

### A. npm 環境警告
- `npm warn Unknown env config "http-proxy"` が実行時に表示される。
- これはアプリコード由来ではなく npm 設定値の警告。
- 対処: CI/ローカルの npm config から `http-proxy` を削除 or 正式キー（`proxy`/`https-proxy`）に移行。

### B. フロントエンドのみで、APIキー秘匿の設計が未整備
- 現在はViteのフロント実装中心で、READMEでも`.env.local`にキーを置く手順が記載されている。
- `ANTHROPIC_API_KEY` を直接フロントで扱うと漏洩リスクが高い。
- 対処: 推薦生成は必ずサーバー経由（Express API等）で実行。

## 4) Haiku の無料クレジット検証について

### 事実確認
- Anthropicの公式価格ページには、Haiku 3.5 のトークン課金単価（入力/出力）はあるが、
  「常設の無料クレジット枠が誰でも何ドルまで使えるか」の固定値は明記されていない。
- そのため「無料でどこまで」は、**アカウント状態（trial付与の有無・残高）依存**。

### 今回のテスト結果
- この環境では `ANTHROPIC_API_KEY` 未設定のため、実API課金テストは未実施。
- 代替として、単価ベースの概算式を準備:
  - 1リクエストコスト ≒ `input_tokens * 単価_in + output_tokens * 単価_out`
  - 無料枠内回数 ≒ `free_credit_usd / cost_per_request`

### 実施手順（次アクション）
1. Anthropic Consoleで `Credits / Billing` の実残高USDを確認。
2. スモークテストを10回実行し、平均トークン消費を採取。
3. 上式で「何回分無料か」を算出。
4. Console残高差分と理論値のズレを確認（誤差5%以内を目標）。
